My Own Formulae

Determining Bits to Borrow

Ever wondered how many bits you need to borrow to create the subnets you desire? Here's the secret formula:

2^b = r, where:

  • b = bits to borrow (because stealing bits is the network admin's pastime)
  • r = required subnets (the magic number you're after)

Solving for b is like finding the password to your network's secret club. Remember, if r isn't a power of 2, you might need to round up or reconsider your life choices.

Calculating Usable Hosts

Want to know how many devices you can plug into your freshly baked subnet? Here's how to do it:

  1. Start with the total bits in IPv4, which is 32. (Because the universe runs on 32-bit magic.)
  2. Subtract the new prefix length (your fancy subnet mask) to get the number of host bits, let's call it n.
  3. Plug n into 2^n - 2 to find out how many usable hosts you've got. The "-2" accounts for the network address and broadcast address, because they're like the bouncers of your subnet party; they're there, but you don't want to use them for dancing.

And there you have it, the magic formula for hosting a subnet party where everyone gets an IP!

Subnetting Formulas

Formula Description
2n - 2 Calculates the number of usable hosts in a subnet, where n is the number of host bits.
32 - Subnet Bits Calculates the number of host bits remaining when subnetting. Subnet Bits = Total Bits (32) - Network Bits.
2h Calculates the total number of addresses in a subnet, where h is the number of host bits (including network and broadcast).
Subnet Mask in CIDR Expresses the subnet mask in Classless Inter-Domain Routing (CIDR) notation (e.g., /24).
2n for Subnets Calculates the number of subnets created by borrowing n bits from the host part.
Subnet Addressing Each subnet's range includes a network address (first), usable addresses, and a broadcast address (last).

Basic Subnetworking Concepts

Concept Description
Subnet A logical subdivision of an IP network, designed to improve performance and security.
Subnet Mask A 32-bit number used to differentiate between the network and host portions of an IP address.
CIDR (Classless Inter-Domain Routing) A method for allocating IP addresses and routing IP packets that allows for variable-length subnet masking.
Broadcast Address An IP address used to send data to all hosts on a subnet; the last address in the subnet.
Network Address The first address in a subnet that identifies the subnet itself; all host bits are set to 0.
Usable Hosts The number of valid IP addresses available for hosts within a subnet, calculated as 2h - 2.
VLSM (Variable Length Subnet Masking) A technique that allows subnets of different sizes to be allocated within the same network.
Supernetting A method of combining multiple smaller networks into a larger network to simplify routing.
Host Bits The portion of an IP address that identifies specific devices within a subnet; determined by the subnet mask.

Relevant Networking Terms

Term Definition
IP Address A unique identifier for a device on a network, assigned in either IPv4 or IPv6 format.
Subnetting The practice of dividing a larger network into smaller, more manageable sub-networks.
Classful Addressing A method of dividing IP addresses into classes (A, B, C, etc.) based on fixed boundaries.
Classless Addressing A more flexible way of allocating IP addresses without fixed boundaries, commonly using CIDR.
Default Gateway The router that forwards traffic from a local network to external networks.
DHCP (Dynamic Host Configuration Protocol) A network protocol that automatically assigns IP addresses to devices on a network.
DNS (Domain Name System) A system that translates human-readable domain names into IP addresses.
NAT (Network Address Translation) A technique used to translate private IP addresses to a public IP address for internet access.
Routing The process of selecting paths in a network along which to send data packets.
TCP/IP (Transmission Control Protocol/Internet Protocol) A suite of communication protocols used to interconnect network devices on the internet.
OSI Model A conceptual framework used to understand network communication in seven layers.
Firewall A security system that monitors and controls incoming and outgoing network traffic based on security rules.
VPN (Virtual Private Network) A technology that creates a secure, encrypted connection over a less secure network, such as the internet.
